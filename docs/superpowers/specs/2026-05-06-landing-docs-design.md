# Landing Page + Docs Site — Design

**Date:** 2026-05-06
**Status:** Approved (brainstorming)
**Hosting:** Vercel
**Framework:** Astro Starlight (docs) + custom Astro pages (landing)

## Goal

Ship a single combined site at `frp-operator.vercel.app` that:

1. Sells the project on `/` with a warm/playful landing page.
2. Hosts Diátaxis-shaped docs at `/docs/*` (tutorials, how-to guides, concepts, reference).
3. Auto-generates the reference layer from code (CRDs, Helm values, operator flags) so docs never drift.

## Positioning (drives all copy)

frp-operator gives every Kubernetes Service a routable public IP, regardless of network configuration, by provisioning cheap cloud frps exits and bin-packing tunnels onto them. Direct competitors:

- **inlets-operator** — provisions one VM per Service. No multi-tunnel, no scheduling.
- **chisel-operator** — multi-tunnel but no scheduling, no cost-aware placement.
- **frp-operator** — multi-tunnel, karpenter-style scheduler, cheapest-fit instance selection, pool-level budget caps, drift + auto consolidation.

### Docs (`/docs/*`)

Diátaxis quadrants, sidebar groups in this order:

- **Getting started** *(tutorials — narrative, follow-along)*
  - Install on kind / minikube
  - Install on production cluster (Helm + LB)
  - Your first Tunnel (LocalDocker provider)
  - Your first DigitalOcean exit
- **Guides** *(how-to — recipe per page)*
  - Restrict exposed ports (`allowPorts`)
  - Pin tunnels to specific regions
  - Rotate exits on schedule
  - Cap monthly traffic per pool
  - Multi-pool gold/silver tiers
  - Migrate from raw frps to operator
  - Upgrade CRDs (CRD chart workflow)
  - Set up Prometheus scrape
  - Use NetworkPolicy for metrics isolation
  - Run multi-replica with leader election
- **Concepts** *(explanation — the why)*
  - Four CRDs and karpenter mapping
  - Scheduling: offerings, requirements, cheapest-fit
  - Disruption: drift, emptiness, consolidation, expiration
  - Liveness probe + cordon-via-condition
  - Hash-based drift detection
- **Reference** *(reference — exhaustive, look-up-able)*
  - CRD reference (auto-generated, one page per kind)
  - Helm values (auto-generated)
  - Operator flags + env vars (auto-generated)
  - Disruption reasons + metrics (hand-written)
  - ADRs (`docs/adr/*`)
  - Karpenter conformance

Each guide page ends with a "Reference" sidebar block listing the exact flags/values/CRD fields it touched. Reference pages back-link to guides that use them (manual cross-linking; not auto-generated).

### Compliance checks

- Body contrast: `--ink-base` on `--bg-base` = 16.4:1 (AAA).
- Accent text on cream: `#d97706` on `#fffbeb` = 3.6:1 — sufficient for non-text UI and large text only. Body links use `--ink-base` color with `text-decoration: underline; text-decoration-color: var(--accent)` to satisfy 4.5:1 while still showing the accent.
- Focus ring `--shadow-glow` 4px halo on every interactive element.
- Buttons enforce `min-height: 44px` and 8px gap from neighbors.
- `@media (prefers-reduced-motion: reduce)` zeroes all transitions in `tokens.css`.
- Icons: SVG only via `astro-icon` (Heroicons + Lucide), no emoji.
- Dark mode tested independently: each token overridden, not auto-inverted.

## Auto-generation pipeline

Source of truth for reference docs is code, never markdown. Three generators, all writing into `site/src/content/docs/reference/`. Generated MDX is committed to git so Vercel build needs no Go toolchain.

### 1. CRD reference — `crd-ref-docs`

```bash
$(CRD_REF_DOCS) \
  --source-path=./api/v1alpha1 \
  --source-path=./pkg/cloudprovider/digitalocean/v1alpha1 \
  --source-path=./pkg/cloudprovider/localdocker/v1alpha1 \
  --templates-dir=hack/crd-ref-templates \
  --renderer=markdown \
  --output-path=site/src/content/docs/reference/crds/
```

One MDX per CRD kind (5 files). Custom templates inject Starlight frontmatter (`title`, `description`).

### 2. Helm values — `helm-docs`

```bash
$(HELM_DOCS) \
  --chart-search-root=charts/frp-operator \
  --template-files=../../hack/helm-values-template.gotmpl \
  --output-file=../../site/src/content/docs/reference/helm-values.mdx
```

Reads `values.yaml` comments → MDX table. `values.yaml` already carries human-readable comments in the chart spec. Only the operator chart's values are documented; the CRD chart exposes just `additionalAnnotations` and `additionalLabels` and is documented inline in the install guide instead.

### 3. Flag reference — `hack/gen-flags-doc.go` (new, ~100 LOC)

- Parses `pkg/operator/flags.go` via `go/ast`.
- Walks `LoadConfigFromArgs` body; extracts every `fs.{String,Bool,Int,Float64,Duration}Var` call: name, default, description.
- Cross-references env-var fallbacks in `applyEnv()` to populate the env-var column.
- Emits MDX table: flag, type, default, env var, description.

### Makefile target

```make
.PHONY: site-gen
site-gen: $(CRD_REF_DOCS) $(HELM_DOCS) ## Regenerate reference docs into site/.
	$(CRD_REF_DOCS) \
	  --source-path=./api/v1alpha1 \
	  --source-path=./pkg/cloudprovider/digitalocean/v1alpha1 \
	  --source-path=./pkg/cloudprovider/localdocker/v1alpha1 \
	  --templates-dir=hack/crd-ref-templates \
	  --renderer=markdown \
	  --output-path=site/src/content/docs/reference/crds/
	$(HELM_DOCS) \
	  --chart-search-root=charts/frp-operator \
	  --template-files=../../hack/helm-values-template.gotmpl \
	  --output-file=../../site/src/content/docs/reference/helm-values.mdx
	go run hack/gen-flags-doc.go > site/src/content/docs/reference/flags.mdx
	cp docs/karpenter-conformance.md site/src/content/docs/reference/karpenter-conformance.mdx
	rsync -a --delete docs/adr/ site/src/content/docs/reference/adrs/
```

Tools (`crd-ref-docs`, `helm-docs`) installed via `go install` into `bin/`, version-pinned alongside `controller-gen`.

### Drift gate (CI)

New `site-gen-drift` job in `.github/workflows/lint.yml`:

```yaml
  site-gen-drift:
    name: Site reference drift gate
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with: { go-version-file: go.mod }
      - run: make site-gen
      - run: git diff --exit-code site/src/content/docs/reference/
```

Same model as `helm-crds` drift gate.

## Local dev

```bash
make site-gen                              # one-time / whenever code changes
cd site && pnpm install && pnpm dev        # http://localhost:4321
```

Hot-reload on MDX + Astro component edits. Generated reference docs require re-running `make site-gen` (no auto-watch — would need a Go file watcher; not worth it for v0).

## Out of scope (v0)

- Custom domain — trivial follow-up via Vercel dashboard.
- Algolia DocSearch — defer until content > ~30 pages.
- i18n / versioned docs — single `next` until v1.
- Lighthouse CI gate — manual for v0.
- PR preview UX (e.g. comment bot posting preview link) — Vercel posts these as a status check; sufficient.
- Web Analytics — Vercel has built-in opt-in; defer.
- Logo polish — placeholder wordmark for v0.
- Auto cross-links between guides ↔ reference — manual MDX `<Card>`/links for v0.
- RSS / blog — no blog in scope.
