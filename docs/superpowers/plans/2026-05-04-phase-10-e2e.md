# Phase 10: E2E Suite Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans.

**Goal:** Rebuild `test/e2e/` against the new CRD shapes (ExitPool, ExitClaim, Tunnel, LocalDockerProviderClass) and run on kind + localdocker. Also rebuild `test/utils/` helpers, `config/overlays/e2e/`, and the GitHub Actions workflow. This is the final validation gate — green e2e == ready to merge `karpenter-refactor` to `main`.

**Architecture:** Single Ginkgo suite under `test/e2e/`. `BeforeSuite` builds the operator image, deploys to kind, applies a shared SchedulingPolicy-equivalent (now `ExitPool` + `LocalDockerProviderClass`), waits for operator readiness via dry-run admission probe (CNPG-pattern). Each `Describe` covers one user-visible feature: tunnel lifecycle, scheduling/binpack, scheduler refusal, exitclaim finalizer, reverse-sync, traffic, resilience. Mirrors the v1alpha1-old e2e suite but rewritten against the new types — much of the logic carries over once the CRD names + spec shapes are remapped.

**Spec:** §16 (test scaffolding), §19 (acceptance criteria #7).

**Prerequisites:** Phases 1-9 merged. Operator binary builds + runs.

**End state:**
- `test/e2e/` rebuilt with full Ginkgo suite, all specs pass on kind + localdocker.
- `test/utils/` rebuilt with helpers (`tunnel`, `exitclaim`, `exitpool`, `kubernetes`, `operator`).
- `config/overlays/e2e/` provides docker.sock + kind-friendly tweaks.
- `Makefile` `test-e2e` target restored (currently disabled stub from Phase 1).

## Task 1: test/utils/ helpers

Mostly mechanical port from the prior `test/utils/` directory (deleted in Phase 1). Take inspiration from `git show 7841921:test/utils/...` but rewrite against the new CRD types.

- `kubernetes/apply.go`: `ApplyServerSide(ctx, yaml []byte) error` runs `kubectl apply --server-side --force-conflicts -f -` via os/exec.
- `kubernetes/wait.go`: generic `WaitFor(ctx, fn func() (bool, error), timeout, interval)`.
- `operator/readiness.go`: `WaitForOperatorReady(ctx, c client.Client, timeout)` — dry-run-applies a tiny ExitPool, expects Ready=False but no admission errors. Replaces the v1alpha1-old "wait for webhook" string-match retry.
- `tunnel/tunnel.go`: `Get`, `WaitForPhase(ctx, c, ns, name, phase, timeout)`.
- `exitclaim/list.go`: `List(ctx, c, namespace) ([]ExitClaim, error)` — namespace is irrelevant for cluster-scoped, but kept signature for symmetry.
- `exitclaim/wait.go`: `WaitForCondition(ctx, c, name, condType, status, timeout)`.
- `exitpool/apply.go`: `Apply(ctx, c, pool *v1alpha1.ExitPool)` server-side apply via raw client.

Commit: `test(utils): rebuild helpers against new CRD types`.

## Task 4: test/e2e/e2e_suite_test.go

```go
//go:build e2e

package e2e

import (
    "context"
    "os"
    "path/filepath"
    "testing"
    "time"

    . "github.com/onsi/ginkgo/v2"
    . "github.com/onsi/gomega"

    "k8s.io/apimachinery/pkg/runtime"
    "k8s.io/client-go/kubernetes/scheme"
    ctrl "sigs.k8s.io/controller-runtime"
    "sigs.k8s.io/controller-runtime/pkg/client"

    v1alpha1 "github.com/mtaku3/frp-operator/api/v1alpha1"
    dov1alpha1 "github.com/mtaku3/frp-operator/pkg/cloudprovider/digitalocean/v1alpha1"
    ldv1alpha1 "github.com/mtaku3/frp-operator/pkg/cloudprovider/localdocker/v1alpha1"
    "github.com/mtaku3/frp-operator/test/utils/kubernetes"
    "github.com/mtaku3/frp-operator/test/utils/operator"
)

var (
    suiteCtx context.Context
    k8sClient client.Client
)

func TestE2E(t *testing.T) {
    RegisterFailHandler(Fail); RunSpecs(t, "frp-operator e2e suite")
}

var _ = BeforeSuite(func() {
    suiteCtx = context.Background()
    By("registering CRDs in the scheme")
    Expect(v1alpha1.AddToScheme(scheme.Scheme)).To(Succeed())
    Expect(ldv1alpha1.AddToScheme(scheme.Scheme)).To(Succeed())
    Expect(dov1alpha1.AddToScheme(scheme.Scheme)).To(Succeed())

    By("building and loading the manager image")
    runOrFail("make docker-build IMG=example.com/frp-operator:v0.0.1")
    runOrFail("kind load docker-image example.com/frp-operator:v0.0.1 --name " + os.Getenv("KIND_CLUSTER"))

    By("applying the e2e overlay")
    runOrFail("kubectl apply -k config/overlays/e2e --server-side --force-conflicts")

    cfg := ctrl.GetConfigOrDie()
    var err error
    k8sClient, err = client.New(cfg, client.Options{Scheme: scheme.Scheme})
    Expect(err).NotTo(HaveOccurred())

    By("waiting for the operator to become Ready")
    Expect(operator.WaitForOperatorReady(suiteCtx, k8sClient, 5*time.Minute)).To(Succeed())

    By("applying shared ExitPool + ProviderClass + credentials")
    yaml, err := os.ReadFile(filepath.Join("fixtures", "shared.yaml"))
    Expect(err).NotTo(HaveOccurred())
    Expect(kubernetes.ApplyServerSide(suiteCtx, yaml)).To(Succeed())
})

var _ = AfterSuite(func() {
    By("deleting the e2e overlay")
    _ = runCmd("kubectl delete -k config/overlays/e2e --ignore-not-found --wait=false")
})
```

## Task 5: per-feature spec files

For each, mirror the v1alpha1-old version (deleted in Phase 1) but rewrite against new CRD types. Spec shapes change; orchestration stays similar.

- **tunnel_test.go**: apply `tunnel_basic.yaml` (a Service of `loadBalancerClass: frp-operator.io/frp` + backend Deployment); `Eventually` Tunnel reaches `Phase=Ready`.
- **scheduling_test.go**: two Tunnels on the same Pool → both bind to one ExitClaim. Then narrow Pool's `Frps.AllowPorts` to exclude port 22; apply Service requesting port 22; assert ExitClaim count unchanged + Tunnel stuck `Allocating`.
- **exitclaim_test.go**: delete an ExitClaim → drain happens; bound tunnel's `AssignedExit` clears; container removed from Docker. Webhook ImmutableWhenReady check is a `PIt` (pending — moved to status validation in Phase 9, e2e-spec'd in v1beta1).
- **reverse_sync_test.go**: ExitClaim becomes Ready with PublicIP → Service.Status.LoadBalancer.Ingress[0].IP populated.
- **webhook_test.go**: rename to `cel_validation_test.go` since webhooks are gone; tests CEL validation rejects bad specs (e.g. `weight:101`).
- **traffic_test.go**: real curl from kind control-plane through frps to backend.
- **resilience_test.go**: docker restart frps container → Tunnel recovers within 90s.

Each file ~50-150 lines.

Commits, one per feature spec.

```makefile
test-e2e:
	./hack/setup-kind-and-shared-dir.sh
	KUBECONFIG=/tmp/frp-operator-e2e.kubeconfig KIND_CLUSTER=frp-operator-test-e2e \
		go test -tags=e2e ./test/e2e/ -v -ginkgo.v -timeout=20m; \
	rc=$$?; \
	if [ $$rc -ne 0 ]; then \
		echo "===== operator logs (post-failure) ====="; \

	fi; \
	if [ "$$E2E_KEEP_KIND" != "1" ]; then make cleanup-test-e2e; fi; \
	exit $$rc
```
